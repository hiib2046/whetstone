# 런타임 중립 플러그인 설계

여러 AI 런타임을 지원하는 플러그인의 공통 동작을 한 곳에 유지하고, 런타임 차이만 어댑터에 가둔다. 상세 구조를 이 스킬에 복제하지 않고 Whetstone 가이드를 단일 기준으로 적용한다.

## 입력

- **대상**: 만들거나 점검할 플러그인의 경로. 새 플러그인이면 만들 위치와 이름.
- **작업 유형**: 새로 제작, 기존 플러그인 전환, 읽기 전용 점검, 점검 후 수정 중 하나.
- **지원 런타임**: 지원하려는 런타임 목록. 기존 등록 파일만으로 확정할 수 있으면 그 목록을 사용한다.

대상이나 작업 유형을 요청에서 확정할 수 없으면 사용자에게 묻는다. 제작·전환에 필요한 지원 런타임을 기존 파일과 요청에서 확정할 수 없을 때도 추측하지 않고 묻는다.

## 수행

1. 작업 전에 [`../../guides/runtime-neutral-plugin.md`](../../guides/runtime-neutral-plugin.md)를 처음부터 끝까지 읽는다.
2. 대상이 플러그인이며 작업의 목표가 여러 런타임 지원인지 확인한다. 기존에는 한 런타임만 지원하더라도 여러 런타임을 지원하도록 전환하는 요청이면 적용 대상이다. 프로젝트 전용 하네스이거나 결과도 한 런타임 전용으로 유지하는 작업이면 적용 대상이 아님을 알린다.
3. 작업 유형에 맞춰 진행한다.
   - **새로 제작**: 요청된 기능과 지원 런타임에 필요한 부품만 만든다.
   - **전환**: 기존 동작과 지원 수준을 먼저 파악하고, 공통 핵심을 하나로 모은 뒤 런타임 차이만 어댑터에 남긴다.
   - **읽기 전용 점검**: 파일을 수정하지 않고 가이드 위반과 검증 근거만 보고한다.
   - **점검 후 수정**: 위반을 확인한 뒤 요청 범위 안에서 고치고 검증한다.
4. 파일을 만들거나 수정하는 작업이면 [`../craft-harness/SKILL.md`](../craft-harness/SKILL.md)를 읽고 함께 적용한다. `craft-harness`가 대상 부품을 판별하여 필요한 공통·부품별 가이드를 선택하게 한다. 읽기 전용 점검에는 적용하지 않는다.
5. 런타임별 등록 형식이나 훅 기능을 새로 연결하거나 바꿀 때는 해당 런타임의 현재 공식 문서로 실제 형식을 확인한다. 확인하지 못한 동작은 지원한다고 표시하지 않는다.
6. `runtime-neutral-plugin.md`의 점검 질문을 원칙 블록마다 하나씩 대조한다. 대상에 없는 선택 부품의 질문은 적용 제외로 판정하고, 위반은 수정 작업일 때 바로 고친다.
7. 변경 위험에 비례해 공통 테스트, 어댑터 테스트, 지원 런타임의 스모크 검증을 실행한다. 실행하지 못한 검증과 그로 인해 확정할 수 없는 지원 범위를 보고한다.

## 완료조건

- 대상과 작업 유형, 지원 런타임이 확정되어 있다.
- `runtime-neutral-plugin.md`의 모든 원칙 블록이 통과·위반·적용 제외 중 하나로 판정되어 있다.
- 수정 작업이면 `craft-harness`가 선택한 가이드의 점검 질문까지 대조되어 있다.
- 지원한다고 표시한 런타임은 가이드가 요구하는 관찰 가능한 검증을 통과했다.
- 읽기 전용 점검에서는 대상 파일이 변경되지 않았다.

## 출력

제작·전환·수정 작업은 `craft-harness`와 같은 완료 보고를 사용한다. 산출물에는 변경 파일, 검증 결과, 검증하지 못한 지원 범위를 함께 적고, 자가 점검에는 위반과 수정 내용만 적는다.

```markdown
## 완료 보고
- **산출물**: <변경 파일, 검증 결과, 검증하지 못한 지원 범위>
- **적용한 가이드**: <runtime-neutral-plugin.md와 craft-harness가 선택한 가이드>
- **자가 점검에서 고친 항목**: <위반 → 수정 내용; 없으면 "위반 없음">
```

읽기 전용 점검은 다음 형식으로 보고한다.

```markdown
## 점검 보고
- **대상**: <플러그인 경로>
- **판정**: <통과 | 수정 필요 | 판정 보류>
- **발견**: <가이드 섹션, 근거 파일 또는 검증 결과, 필요한 수정>
- **검증하지 못한 범위**: <없으면 "없음">
```

## 입출력 예시

**입력 — 전환:** “`plugins/reviewer`를 Claude Code와 Codex에서 함께 쓰는 런타임 중립 플러그인으로 바꿔줘.”

**출력:**

```markdown
## 완료 보고
- **산출물**: 공통 스킬 본문과 Codex·Claude Code 연결 파일을 정리하고 관련 테스트를 통과함
- **적용한 가이드**: runtime-neutral-plugin.md, common-guide.md, skill-guide.md
- **자가 점검에서 고친 항목**: 런타임별로 복제된 스킬 본문 → 공통 skills/로 통합
```

**입력 — 읽기 전용 점검:** “`plugins/reviewer`가 Claude Code·Codex·Hermes에서 런타임 중립 구조를 지키는지 파일을 고치지 말고 점검해줘.”

**출력:**

```markdown
## 점검 보고
- **대상**: plugins/reviewer
- **판정**: 수정 필요
- **발견**: 공통 핵심과 어댑터의 경계 — Codex 어댑터가 공통 검증 로직을 복제함; 공통 스크립트로 이동 필요
- **검증하지 못한 범위**: Hermes 실행 환경이 없어 스모크 검증하지 못함
```
