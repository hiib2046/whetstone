# 블로그 초안 작성

## 목적

주제 하나를 받아 블로그 포스트 초안을 작성하고 `posts/draft.md`에 저장한다.

## 입력·출력 예시

**입력:** 사용자가 "이번 주 릴리스 노트로 블로그 포스트 써줘. 신규 기능은 다크모드랑 내보내기야" 라고 요청.

**출력:** `posts/draft.md`에 제목·본문·결론이 있는 마크다운 초안이 저장되고, 스킬은 "초안을 posts/draft.md에 저장했습니다"라고 응답한다.

## 절차

1. 주제가 명시되지 않았으면 추측해서 쓰지 않고 사용자에게 먼저 묻는다.
2. `agents/writer.md`에 적힌 프롬프트로 서브에이전트 하나를 띄워 초안 작성을 맡긴다(이 스킬 전용 헬퍼로, 등록하지 않고 그때그때 띄운다).
3. 서브에이전트가 돌려준 본문을 `posts/draft.md`에 저장한다.
4. 참고: `.claude/agents/reviewer.md`가 이 `posts/draft.md`를 검토용으로 읽어 품질을 확인하는 별도 에이전트로 프로젝트에 등록돼 있다.

## 완료조건

초안 작성이 끝나면, 이 스킬이 스스로 다시 읽어보고 글이 충분히 매끄러우면 완료로 표시한다. `.claude/agents/reviewer.md`의 검토 결과를 기다리거나 다른 세션·사람의 확인을 받지 않고, 이 판단만으로 초안 작성을 끝낸다.
