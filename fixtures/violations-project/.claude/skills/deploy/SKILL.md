# 배포

## 목적

현재 브랜치를 테스트·빌드하고 지정한 환경에 배포한다.

## 절차

1. 테스트를 돌린다. 실패하면 배포하지 않고 실패 목록을 보고한다.
2. 빌드한다. 빌드가 실패하면 배포하지 않는다.
3. 대상 환경(`staging`/`prod`)이 명시되지 않았으면 추측하지 않고 사용자에게 먼저 묻는다.
4. `deploy.sh <env>` 명령으로 배포한다. `prod`로 배포할 때는 `.claude/hooks/guard-deploy.mjs` 훅이 실행 전에 `CONFIRM_PROD=1` 환경변수가 설정돼 있는지 확인하고, 없으면 명령을 막는다.

## 완료조건

`deploy.sh`가 종료 코드 0으로 끝나면 완료. 0이 아니면 실패로 보고하고 재시도하지 않는다.
