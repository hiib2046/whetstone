# AGENTS.md

이 저장소에서 작업할 때 지키는 규칙.

## 테스트

- 코드를 변경하면 `pytest tests/`를 실행하고 통과를 확인한 뒤 커밋한다.
- 실패하는 테스트를 건드리지 않고 방치하지 않는다 — 원인을 고치거나 스킵 사유를 커밋 메시지에 남긴다.

## 스타일

- 새 함수에는 타입 힌트를 단다.
- 큰 파일은 건드리지 마라.

## 커밋

- 커밋 메시지는 한 줄 요약 + 필요하면 본문.
- `main` 브랜치에 직접 푸시하지 말고 PR을 통해 병합한다 — 리뷰 없이 들어간 변경은 되돌리기 비용이 커서, 병합 전에 한 번은 다른 눈이 보게 한다.
