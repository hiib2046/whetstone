# 로그 집계 → out/data.json

목적: 원시 로그를 이후 단계(요약 문서 작성 등)가 바로 읽어 쓸 수 있는 구조화된 형태로 바꿔 둔다. 집계 결과는 이 스킬의 세션이 끝난 뒤에도 다른 도구가 읽어야 하므로 대화 안에 담지 않고 파일로 남긴다.

## 절차

1. `logs/` 디렉토리의 `*.log` 파일을 모두 읽는다. `logs/`가 없거나 비어 있으면 집계하지 말고 사용자에게 알린다.
2. 로그 한 줄마다 카테고리(로그 레벨: ERROR/WARN/INFO)를 추출해 개수를 센다. 이 세 레벨에 해당하지 않는 줄(DEBUG, TRACE, 레벨 표기가 없는 줄 등)은 집계에서 제외한다 — 추측으로 셋 중 하나에 넣지 않는다.
3. 전체 합계와 카테고리별 개수를 아래 스키마로 `out/data.json`에 쓴다(기존 파일이 있으면 덮어쓴다).

## 입력·출력 예시

**정상 집계**

입력: `logs/app.log` 4줄 — ERROR 1, WARN 1, INFO 1, DEBUG 1(레벨 아님, 집계 제외)

출력(`out/data.json`):

```json
{
  "total": 3,
  "byCategory": { "ERROR": 1, "WARN": 1, "INFO": 1 }
}
```

**`logs/`가 없거나 비어 있음**

입력: `logs/` 디렉토리 자체가 없음

출력: `out/data.json`을 쓰지 않고, "집계할 로그 없음"을 사용자에게 알리고 종료.

## 완료조건

`out/data.json`이 `total`·`byCategory` 키를 모두 포함해 저장되면 완료. `logs/`에 로그가 하나도 없으면 파일을 만들지 않고 "집계할 로그 없음"을 알리고 종료.
