# answers.md — similar-names-project

## ① 기대 탐지 결과

- 하네스로 묶이지 않음. `blog-writer`와 `blog-checker`는 이름(`blog-` 접두)과 주제(블로그)만 유사할 뿐, 서로를 부르는 참조도 없고(문서 참조·서브에이전트 위임·코드 호출 없음), 공유하는 파일 경로도 없다(둘 다 고정 경로를 쓰지 않고 그때그때 사용자가 주는 입력을 받음).
- 스펙 "하네스 묶기" 2번 규칙에 따라 **"관련 추정"**으로 표시돼야 한다. 근거는 "이름뿐, 연결 흔적 없음" — 직접 참조도 공유 상태도 발견되지 않았다는 사실을 명시해야 한다.
- 억지로 하나의 하네스로 묶으면 실패(오탐). 완전히 무관한 외톨이 2개로 각각 분리해 버려도 스펙 취지(관련 추정 표시)에 못 미치므로 부족 — "관련 추정" 탭/표시가 별도로 있어야 한다.

## ② 심은 위반

없음. 두 파일 모두 가이드 위반 없이 건전하게 작성됨(common-guide·skill-guide의 이름·description·절차 앞 방향·완료조건 검증 가능·입출력 예시·빈칸 처리 등을 모두 충족). 리포트가 이 두 파일에서 위반을 만들어내면(오탐) 실패로 판정한다.

## ③ 기대 리포트 행동

"관련 추정" 전용 탭(또는 요약 탭의 관련 추정 섹션)에 `blog-writer`·`blog-checker`가 한 쌍으로 표시되고, 옆에 근거 문구(예: "이름 유사 — blog- 접두어 공유, 직접 참조·공유 파일 없음")가 붙어야 한다. 두 스킬 모두 개선 리스트에는 위반 항목이 없어야 한다(빈 리스트 또는 "위반 없음" 표시).
