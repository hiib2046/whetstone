# 블로그 글 검토

목적: 발행 직전 글에서 사람이 놓치기 쉬운 오탈자·문법 오류·문체 불일치를 찾아 고칠 지점을 짚어준다. 문장을 대신 다시 쓰지 않고 지점만 짚는다 — 글의 목소리는 원저자의 것으로 남겨야 하기 때문이다.

## 절차

1. 점검할 텍스트를 받는다 — 사용자가 대화에 붙여넣거나 파일 경로를 알려준다. 텍스트도 경로도 없으면 추측하지 말고 사용자에게 묻는다.
2. 문장 단위로 오탈자·조사 오류를 찾는다.
3. 문체(존댓말/반말, 문장 종결 어미)가 글 안에서 섞여 있는지 확인한다.
4. 발견한 항목을 `줄/문장 인용 → 문제 → 수정 제안` 형식으로 나열해 돌려준다.

## 입력·출력 예시

입력: "재택근무는 자유롭다. 하지만 시간관리가 안되면 오히려 더 피곤해요."

출력:

```
1. "시간관리가" → "시간 관리가" (띄어쓰기)
2. "피곤해요" → 앞 문장이 평서체("자유롭다")인데 뒤 문장만 존댓말("피곤해요") — 문체 통일 필요
```

## 완료조건

텍스트를 문장 단위로 훑어 오탈자·문체 불일치 항목을 모두 나열하면 완료. 발견된 문제가 없으면 "문제 없음"을 명시해 완료.
