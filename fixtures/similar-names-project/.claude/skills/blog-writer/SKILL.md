# 블로그 글 초안 작성

목적: 사용자가 준 주제·개요를 사람이 바로 다듬어 발행할 수 있는 수준의 글 초안으로 바꾼다. 새로 조사하지 않고, 사용자가 준 정보 안에서 서론-본론-결론의 짜임을 갖춘다.

## 절차

1. 사용자에게 받은 주제와 개요(있으면)를 확인한다. 개요가 없으면 추측해서 채우지 말고, 다루고 싶은 핵심 포인트 2~3개를 먼저 묻는다.
2. 핵심 포인트를 본론 섹션 순서로 배열한다 — 순서가 논리적으로 이어져야 독자가 안 헤맨다.
3. 서론(문제 제기)·본론(포인트별 섹션)·결론(요약+행동 제안) 구조로 초안을 쓴다.
4. 사용자에게 마크다운 텍스트로 초안 전체를 돌려준다(파일로 저장하지 않는다 — 저장 여부는 사용자 판단).

## 입력·출력 예시

입력: 주제 "재택근무 생산성", 핵심 포인트 ["시간 관리", "커뮤니케이션 도구"]

출력:

```markdown
# 재택근무에서 생산성을 지키는 법

재택근무는 자유로운 만큼...(서론)

## 시간 관리
...

## 커뮤니케이션 도구
...

## 마무리
...
```

## 완료조건

서론·본론(핵심 포인트별 섹션)·결론이 모두 있는 마크다운 초안을 반환하면 완료. 핵심 포인트 중 하나라도 섹션이 빠지면 미완료.
