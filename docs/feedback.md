# 피드백·이슈

버그·개선점은 **GitHub Issues**(github.com/hiib2046/whetstone/issues)에 모은다 — 로컬 파일이 아니라. 다른 프로젝트에서 whetstone을 쓰다 발견해도 `gh issue create -R hiib2046/whetstone`로 **어느 폴더에서든** 등록되고, 어느 사용자 로컬도 더럽히지 않기 때문이다.

**플러그인은 사용자 공간에 아무것도 쓰지 않는다.** 스킬이 실행 중 이슈·피드백을 파일로 남기지 않는다 — 설치본은 업데이트 시 지워지는 캐시라 신뢰할 수 없고, 사용자 프로젝트를 오염시킨다.
